# VisionMark 落地页操作文档

## 1. 项目说明

当前网站是基于 `Next.js 16 + App Router + Tailwind CSS v4` 搭建的单页落地页，已经从原始英文 SaaS 模板改成了 VisionMark 的中文宣传页。

当前页面主定位：

- VisionMark 是一个面向视频场景的综合 AI 浏览器插件
- 核心叙事是“先理解视频，再辅助用户观看视频”
- 页面语言已切换为简体中文

## 2. 常用启动命令

在项目根目录执行：

```powershell
npm run dev
```

本地构建检查：

```powershell
npm run build
```

代码检查：

```powershell
npm run lint
```

## 3. 关键文件说明

### 3.1 页面主文件

文件：

`src/app/page.tsx`

作用：

- 首页所有可见内容都在这里
- 包含顶部导航、首屏、问题区、能力说明区、FAQ、页脚

### 3.2 全局布局与站点信息

文件：

`src/app/layout.tsx`

作用：

- 配置网站标题和描述
- 配置全局字体
- 设置页面语言为 `zh-CN`

### 3.3 全局样式

文件：

`src/app/globals.css`

作用：

- 配置 Tailwind 主题变量
- 把正文和标题字体绑定到自定义中文字体

### 3.4 图片资源

目录：

`public/`

作用：

- 存放首页插画图片
- 页面中通过 `/文件名.png` 的方式引用

### 3.5 Next 开发配置

文件：

`next.config.ts`

当前已配置：

- `devIndicators: false`

作用：

- 关闭开发环境左下角的 Next.js 小球提示器

## 4. 当前页面结构

### 4.1 顶部导航

位于 `page.tsx` 顶部 `header` 中。

当前导航文案：

- 为什么需要
- 它如何工作
- 常见问题

当前右侧按钮：

- 立即安装

### 4.2 首屏 Hero

当前内容：

- 标题：`先理解视频，再更高效地观看`
- 副标题：介绍 AI 总结、重点提炼、知识点、热词解释与片段跳过
- 按钮：`立即安装`

首屏左右两张插画的位置也在这个区块里控制。

### 4.3 为什么需要 VisionMark

这一部分的数据来自 `problemCards` 数组。

当前三张卡片分别是：

- 长视频难快速看懂
- 重点与上下文分散
- 低价值片段打断体验

如果以后要新增、删减或改图，优先改这个数组。

### 4.4 它如何工作

左侧是 4 个折叠项，当前逻辑是：

- AI 先理解视频内容
- 输出结构化结果
- 在观看过程中实时辅助
- AI 分析与社区标注持续优化

右侧是演示框，目前按你的要求保留了原模板的 mock 样式，没有改成 VisionMark 的业务卡片。

### 4.5 常见问题

当前 FAQ 包含 4 个问题：

- VisionMark 是什么？
- 它只是跳过工具吗？
- 目前能提供哪些能力？
- 社区标注有什么作用？

### 4.6 页脚

页脚分为两部分：

- 上半部分：订阅产品更新
- 下半部分：底部导航和版权信息

当前年份使用自动获取，不需要手动修改年份。

## 6. 如何修改图片

### 6.1 更换首屏左上 / 右下插画

位置：

`src/app/page.tsx`

搜索：

`const heroIllustrations = {`

可以修改：

- `src`
- `alt`

如果要调整首屏插画位置，继续看 Hero 下方这两个绝对定位容器：

- 左图容器带有 `-left-16 -top-2`
- 右图容器带有 `bottom-0 right-[-3.75rem]`

改这些偏移值即可。

### 6.2 更换问题区三张图片

位置：

`src/app/page.tsx`

还是在 `problemCards` 数组中修改：

- `image`
- `imageClassName`

其中 `imageClassName` 控制图在卡片中的显示宽度。

### 6.3 更换 FAQ 左侧大图

位置：

`src/app/page.tsx`

搜索：

`/ScreenShot_2026-03-12_181656_743.png`

直接改成新的图片路径即可。

### 6.4 新增图片资源

步骤：

1. 把图片放进 `public/`
2. 在 `page.tsx` 中通过 `/图片名.png` 引用
3. 建议英文文件名或无空格文件名，避免路径问题

## 7. 如何修改字体

字体配置在：

`src/app/layout.tsx`

当前使用：

- 正文：`Noto Sans SC`
- 标题：`Noto Serif SC`
- 等宽：`Geist Mono`

字体变量绑定在：

`src/app/globals.css`

当前映射：

- `--font-sans: var(--font-noto-sans-sc)`
- `--font-serif: var(--font-noto-serif-sc)`

如果要换字体，通常需要同时改这两个文件。

## 8. 当前已做的特殊处理

### 8.1 年份自动更新

位置：

`src/app/page.tsx`

使用：

```tsx
const currentYear = new Date().getFullYear();
```

页脚版权不会再写死 2024 或 2025。

### 8.2 左下角英文小球已关闭

位置：

`next.config.ts`

配置：

```ts
devIndicators: false
```

### 8.3 页眉不是毛玻璃

当前页眉已经改成：

- 实心白底
- 无毛玻璃
- 比原模板略窄

相关代码在 `page.tsx` 的 `header` 区块。

## 9. 当前需要注意的点

### 9.1 “它如何工作”右侧演示框仍是模板样式

这是按当前需求保留的。

如果后续要改成更像 VisionMark 产品的真实演示卡片，可以改 `solution` 区块右侧那个白色 mock 盒子。

### 9.2 “立即安装”按钮目前还是占位链接

当前按钮链接仍然是：

`#getstarted`

如果后续有真实插件下载地址或 Chrome 商店地址，需要统一替换顶部按钮和首屏按钮的 `href`。

## 10. 推荐修改流程

每次改完页面后，建议按下面顺序检查：

1. 修改代码
2. 运行 `npm run lint`
3. 运行 `npm run build`
4. 本地打开页面检查中文换行、图片位置和按钮文案
